## Fix 1 — News photos (seed online images + serializer passthrough)
**Cause:** the seed stores fake storage paths (`news/…jpg`) that don't exist, and `NewsController` wraps them in `Storage::url()` → a relative `/storage/…` the app can't fetch.
- **`app/Http/Controllers/Api/V1/NewsController.php`:** change the `image_url` mapping so a **full URL passes through** unchanged: `image_url => $post->image_path ? (\Illuminate\Support\Str::startsWith($post->image_path, ['http://','https://']) ? $post->image_path : Storage::url($post->image_path)) : null`. (App-one-safe: any storage-path news still goes through `Storage::url`.)
- **`database/seeders/AppTwoDemoSeeder.php`:** set the three news posts that currently have an `image_path` to **stable online sample image URLs** — use football/stadium shots, e.g. `https://images.unsplash.com/photo-<id>?auto=format&fit=crop&w=800&q=60`, or `https://picsum.photos/seed/<slug>/800/450` as a guaranteed-up fallback. Give three **distinct** images; **leave the fourth post's `image_path` null** (it demos the no-image layout). Keep everything else idempotent.
- Mobile already has an `errorBuilder` on `Image.network` (Home + detail) — no change needed there, but **verify** a failed load degrades gracefully (no layout break).

## Fix 5 — Full bilingual fixtures/results (locked: full bilingual)
**Cause:** `_ResultCard` hardcodes the English word **"Lusail"**, and `fixtures` has **no Arabic** for `opponent`/`competition` (no `_ar`, no `HasLocalizedContent`), so both stay Latin in the AR view.
- **Migration** (new): add nullable `opponent_ar` and `competition_ar` to `fixtures` (after `opponent`/`competition`). Additive; app-one fixtures leave them null → fall back to base (English) → app one unchanged.
- **`app/Models/Fixture.php`:** `use HasLocalizedContent;` and add `opponent_ar`, `competition_ar` to `#[Fillable]`. (Keep `ScopedToApp` etc.)
- **`app/Http/Controllers/Api/V1/MatchController.php`:** serialize `'opponent' => $fixture->localized('opponent')` and `'competition' => $fixture->localized('competition')` (respects `Accept-Language`, falls back to base). Everything else (scores, dates, ids) unchanged.
- **Seed (`AppTwoDemoSeeder`):** use **real QSL clubs** as opponents so the Arabic is authentic, and set `opponent_ar` + `competition_ar` on every app-two fixture (results + upcoming). Suggested opponents + AR: Al Sadd `السد`, Al Rayyan `الريان`, Al Duhail `الدحيل`, Al Gharafa `الغرافة`, Al Arabi `العربي`, Al Wakrah `الوكرة`, Umm Salal `أم صلال`, Qatar SC `نادي قطر`. Competitions: `Qatar Stars League` → `دوري نجوم قطر`, `Amir Cup` → `كأس الأمير`. **Align the `standings` `club_name`/`club_name_ar` to the same real QSL clubs** (with Lusail SC `نادي لوسيل` as own-club) so the AR table + results read as one coherent league.
- **Mobile `_ResultCard`** (`matches_screen.dart`): replace the hardcoded string `'Lusail $score ${match.opponent}'` with a **localized, RTL-safe scoreline** — a `Row` of `[clubName, score, opponent]` where `clubName` = a new l10n `clubShortName` (`Lusail` / `لوسيل`) and `opponent` is the now-localized `match.opponent`; render the score (`ourScore–opponentScore`) between them. Because it's a `Row` (not a concatenated string), it **mirrors correctly under RTL** and keeps Lusail's score adjacent to Lusail (semantically correct). `match.competition` is now localized server-side, so the existing competition `Text` needs no change beyond receiving the localized value. Use an en-dash `–` for the score.
