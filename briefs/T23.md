## Deliverables

### 1. `Account` model — parse the discount fields (T19/T21 payload)
`lib/src/features/auth/models/account.dart`: add `discountPercent` (from `discount_percent`, `num?→double`, default 0) and `discountCapPercent` (from `discount_cap_percent`, default 0). (The `membership` block is T24 — don't add it here.) These already come back on `/me`; parsing them is additive and harmless for app one (app-one accounts return `discount_percent: 0`).

### 2. Self-registration in the auth layer
- `AuthRepository.register(...)` (`features/auth/data/auth_repository.dart`): `POST /auth/register` with `{name, email, password, password_confirmation, phone?}` → parse `LoginResponse` (`{token, parent}`, same shape as `login`), `tokenStorage.writeToken(result.token)`, return it. Wrap in the existing `try/catch → ApiException.fromObject` pattern.
- `SessionController.register({name, email, password, passwordConfirmation, phone})` (`features/session/session_controller.dart`): mirror `login()` — set busy, call `repository.register(...)`, then `getMe()`, set `SessionStatus.authenticated`; on error reset to `unauthenticated` + `rethrow` (so the screen shows field/validation errors). Backend `register` returns 422 with `email`/`password` errors → surfaced via `ApiException.firstErrorFor(...)` like `LoginScreen`.

### 3. `MemberAuthScreen` — app-two login + register (replaces the T22 stub)
New screen in the content/supporter presentation folder. **Model it on the app-one `LoginScreen`** (the branded hero `_Header` with `brand.heroLogo`, `Form`, `ApiException` field errors, busy spinner, `_friendlyErrorMessage`) but **app-two-appropriate**: a **Sign in / Create account toggle** (SegmentedButton or a tab) —
- **Sign in**: email + password → `sessionController.login(...)`.
- **Create account**: name + email + password + confirm-password + phone (optional) → `sessionController.register(...)`; validate email format, password min 8, confirm matches.
- **No** accept-invite link and **no** staff-scanner entry (those are app-one only).
Wire T22's `MembershipGateScreen` "Sign in" button (currently a `// TODO(T23)` stub) to push/show this screen. On success the session becomes authenticated and the Membership tab re-renders to the wallet (no manual navigation needed — it's `ref.watch(sessionControllerProvider)`-driven).

### 4. Extract a shared `QrPassCard` widget (reuse the rotating QR)
The member wallet needs the same rotating QR the app-one parent shows (a member **is** a `ParentAccount`, so `/scan-token` via `parentScanRepositoryProvider.fetchParentToken()` already works). Extract the QR pass — the stateful token fetch + `QrImageView` + countdown + refresh + `WidgetsBindingObserver` lifecycle refresh — from `QrScreen` into a reusable **`QrPassCard`** widget (in `theme/widgets/` or `scan/presentation/`), and have the app-one `QrScreen` render `QrPassCard` beneath its existing title/subtitle. **Preserve the existing widget keys `parent-qr` + `qr-countdown` and QrScreen's current appearance** so the app-one QrScreen test stays green. `QrPassCard` should take optional `title`/`subtitle` (default null) so the member wallet can pass member-appropriate copy. *(If extraction risks the app-one test, the acceptable fallback is a small dedicated member QR widget that reuses `parentScanRepositoryProvider` — but prefer the shared extraction.)*

### 5. `DiscountWalletScreen` (member) + role routing
Replace the authed branch of the Membership tab with role routing:
- `account_type == 'member'` → **`DiscountWalletScreen`**: a scrollable, pull-to-refresh screen with —
  1. a **broadcast-style discount card** (echo the app-one points-hero look — big `discountPercent`% with a gold/navy treatment) + caption "redeemable toward academy registration" and a small "of {discountCapPercent}% max" note;
  2. the **`QrPassCard`** with member copy ("Show this at the match to earn 0.5%");
  3. a **discount history** list — reuse `PlayerRepository.fetchAccountTransactions('/me/transactions')` (already exists) + the app-one transaction model; **format each `points` value as a percentage (`points / 100`, e.g. `+0.5%`)** since a member's ledger is in percent-basis-points, not points. Loading/empty/error via the existing `.when` + `ApiException` pattern.
  - **Pull-to-refresh** calls `sessionController.refreshAccount()` (updates `discountPercent`) and invalidates the history provider.
- `account_type == 'vvip_member'` → a simple placeholder card ("Your VVIP membership card — coming soon") — T24 replaces it.

### 6. Logout in the authed shell
The `SupporterShell`'s `BrandAppBar` should expose **logout** when authenticated (reuse `sessionController.logout()`), matching app one's header actions. Guests see no logout. (Keep the theme + language toggles.)

### 7. l10n (`lfc_core/lib/l10n/app_en.arb` + `app_ar.arb`, key parity, MSA Arabic)
Add keys for: sign-in/create-account toggle, name/confirm-password labels + "passwords don't match", "Create account" button, wallet title, "{n}% discount" / "redeemable toward academy registration" / "of {cap}% maximum", member QR title+subtitle, "Discount history" + empty/entry formatting, "Sign out", and the VVIP "coming soon" placeholder. Regenerate `AppLocalizations` (`flutter gen-l10n`). RTL must render correctly.
